# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in any of my repositories, please report it responsibly.

**Please do NOT open a public issue.**

Instead, send an email to the maintainer or use [GitHub's private vulnerability reporting](https://docs.github.com/en/code-security/security-advisories/guidance-on-reporting-and-managing-vulnerabilities/privately-reporting-a-security-vulnerability) on the affected repository.

### What to include

- A description of the vulnerability
- Steps to reproduce the issue
- Any potential impact

I will acknowledge your report within 48 hours and aim to release a fix as soon as possible.

## Supported Versions

Only the latest version on the `main` branch of each repository is actively supported.
